Highway is a C++ library that lets you write SIMD code once and run it optimally on any CPU. Write your algorithm using Highway's portable ops, and Highway automatically selects the best available instruction set at runtime — from SSE2 to AVX-512 on x86, NEON to SVE2 on ARM, and beyond.

## Why Highway?

Highway gives you **5–10× speedups** over scalar code with **5× lower energy usage** by mapping directly to CPU SIMD instructions — no auto-vectorization guesswork.

## Get started

<Steps>
  <Step title="Install Highway">
    Add Highway to your project. The fastest path is via a package manager:
    ```bash
    # Debian/Ubuntu
    sudo apt install libhwy-dev

    # vcpkg
    vcpkg install highway
    ```
    See [Installation](/installation) for CMake, Bazel, and Git submodule instructions.
  </Step>
  <Step title="Write your vector function">
    Create a `.cc` file with your SIMD logic inside `HWY_NAMESPACE`:
    ```cpp
    #include "hwy/highway.h"
    HWY_BEFORE_NAMESPACE();
    namespace project {
    namespace HWY_NAMESPACE {
    namespace hn = hwy::HWY_NAMESPACE;

    void MultiplyAdd(const float* HWY_RESTRICT a,
                     const float* HWY_RESTRICT b,
                     const float* HWY_RESTRICT c,
                     float* HWY_RESTRICT out, size_t count) {
      const hn::ScalableTag<float> d;
      const size_t N = hn::Lanes(d);
      for (size_t i = 0; i < count; i += N) {
        auto va = hn::Load(d, a + i);
        auto vb = hn::Load(d, b + i);
        auto vc = hn::Load(d, c + i);
        hn::Store(hn::MulAdd(va, vb, vc), d, out + i);
      }
    }

    }  // namespace HWY_NAMESPACE
    }  // namespace project
    HWY_AFTER_NAMESPACE();
    ```
  </Step>
  <Step title="Enable dynamic dispatch">
    Add three lines to compile your function for all available targets and dispatch at runtime:
    ```cpp
    #define HWY_TARGET_INCLUDE "your_file.cc"
    #include "hwy/foreach_target.h"
    #include "hwy/highway.h"
    // ... your HWY_NAMESPACE code ...
    HWY_EXPORT(MultiplyAdd);
    // Call the best implementation for this CPU:
    HWY_DYNAMIC_DISPATCH(MultiplyAdd)(a, b, c, out, count);
    ```
  </Step>
  <Step title="Build with optimizations">
    Compile with `-O2` or higher. Highway inlines aggressively — without optimization, SIMD code will be slow.
    ```bash
    cmake -DCMAKE_BUILD_TYPE=Release ..
    make -j
    ```
  </Step>
</Steps>
