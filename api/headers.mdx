title: "Headers"
description: "All public Highway headers, what they provide, and when to include each one."
---

Highway's public API is split across several headers. Include only what you need: most SIMD code needs only `hwy/highway.h`, while utilities for allocation, cache control, and benchmarking are available in standalone headers that can be included without pulling in the full vector machinery.

<Note>
Headers named `*-inl.h` must be included **inside** `HWY_NAMESPACE` (i.e., between `HWY_BEFORE_NAMESPACE()` and `HWY_AFTER_NAMESPACE()`). They are re-included once per compilation target.
</Note>

---

## hwy/highway.h

The main Highway header. Include it from any source file or header that uses vector types or calls Highway ops.

```cpp
#include "hwy/highway.h"
```

Including `hwy/highway.h` from a header (as opposed to a `.cc` file) is allowed — it enables declaring functions implemented out of line — but may increase compile times. The header transitively includes `hwy/base.h`, target detection, and all op definitions for the current target.

**Typical usage:**

```cpp
#include "hwy/highway.h"

HWY_BEFORE_NAMESPACE();
namespace my_project {
namespace HWY_NAMESPACE {

namespace hn = hwy::HWY_NAMESPACE;

void ScaleArray(const float* HWY_RESTRICT in, float* HWY_RESTRICT out,
                size_t count, float scale) {
  const hn::ScalableTag<float> d;
  const auto vscale = hn::Set(d, scale);
  const size_t N = hn::Lanes(d);
  for (size_t i = 0; i < count; i += N) {
    hn::Store(hn::Mul(hn::Load(d, in + i), vscale), d, out + i);
  }
}

// NOLINTNEXTLINE(google-readability-namespace-comments)
}  // namespace HWY_NAMESPACE
}  // namespace my_project
HWY_AFTER_NAMESPACE();
```

## hwy/base.h

Compiler and platform definitions that do not require the full Highway vector machinery. Include this from headers that only need low-level utilities such as `PopCount`, `Unpredictable1`, type traits, or the `HWY_*` compiler-detection macros.

```cpp
#include "hwy/base.h"
```

`hwy/base.h` provides:

- `HWY_MAJOR`, `HWY_MINOR`, `HWY_PATCH` — version macros
- `HWY_VERSION_GE(major, minor)` / `HWY_VERSION_LT(major, minor)` — version comparisons
- Compiler and architecture detection macros (`HWY_COMPILER_CLANG`, `HWY_ARCH_X86`, etc.)
- `PopCount(x)` — population count (number of set bits)
- `Unpredictable1()` — returns 1 without the compiler knowing it, useful for preventing dead-code elimination in benchmarks

**Example:**

```cpp
#include "hwy/base.h"

// Use compiler detection without pulling in vector ops.
#if HWY_ARCH_X86
  // x86-specific path
#endif

uint32_t CountSetBits(uint64_t x) {
  return hwy::PopCount(x);
}
```

## hwy/foreach_target.h

Enables dynamic dispatch by re-including your translation unit once per enabled target. Each re-inclusion compiles the same source code for a different instruction set, placing the result in a unique target-specific namespace.

```cpp
// Must come before any inclusion of hwy/highway.h.
#define HWY_TARGET_INCLUDE "path/to/my_kernel-inl.h"
#include "hwy/foreach_target.h"
#include "hwy/highway.h"
```

<Warning>
`hwy/foreach_target.h` must be included **before** `hwy/highway.h`. Define `HWY_TARGET_INCLUDE` to the path of your `-inl.h` file before including this header.
</Warning>

After `foreach_target.h` has compiled all targets, use `HWY_EXPORT` and `HWY_DYNAMIC_DISPATCH` to declare and call the best available implementation at runtime:

```cpp
// my_kernel.cc
#define HWY_TARGET_INCLUDE "my_kernel-inl.h"
#include "hwy/foreach_target.h"
#include "hwy/highway.h"

// Declare the dispatch table.
HWY_EXPORT(ScaleArray);

// Entry point — calls the best available target at runtime.
void ScaleArray(const float* in, float* out, size_t n, float scale) {
  HWY_DYNAMIC_DISPATCH(ScaleArray)(in, out, n, scale);
}
```

## hwy/aligned_allocator.h

Provides aligned memory allocation suitable for use with `Load` and `Store`. Vectors require alignment to at least `HWY_ALIGNMENT` bytes (128 bytes, chosen to prevent false sharing and match L2 prefetcher granularity on several microarchitectures).

```cpp
#include "hwy/aligned_allocator.h"
```

Key functions and types:

| Name | Description |
|------|-------------|
| `AllocateAligned<T>(n)` | Allocates `n` elements of type `T` with `HWY_ALIGNMENT` alignment. Returns `AlignedFreeUniquePtr<T[]>` (automatically freed on destruction). |
| `AllocateAlignedBytes(n)` | Raw byte allocation, aligned to `HWY_ALIGNMENT`. |
| `FreeAlignedBytes(ptr, ...)` | Frees memory allocated by `AllocateAlignedBytes`. |
| `MakeUniqueAligned<T>(args...)` | Aligned equivalent of `std::make_unique<T>`. |
| `AlignedVector<T>` | `std::vector<T, AlignedAllocator<T>>` — STL-compatible aligned vector. |

**Example:**

```cpp
#include "hwy/aligned_allocator.h"
#include "hwy/highway.h"

namespace hn = hwy::HWY_NAMESPACE;

void Example(size_t count) {
  const hn::ScalableTag<float> d;
  // Allocate aligned storage for count floats.
  auto buf = hwy::AllocateAligned<float>(count);

  // buf.get() is aligned to HWY_ALIGNMENT bytes — safe for Load/Store.
  auto v = hn::Load(d, buf.get());
  // ...
}
```

<Note>
Because `Lanes(d)` may not be a compile-time constant on scalable vector targets, do not use it as an array dimension. Always allocate dynamically via `AllocateAligned`.
</Note>

## hwy/cache_control.h

Standalone cache-control functions that are independent of the vector op machinery. You can include this header without including `hwy/highway.h`.

```cpp
#include "hwy/cache_control.h"
```

| Function | Description |
|----------|-------------|
| `Prefetch(const T* p)` | Begins loading the cache line containing `p` to reduce latency of subsequent loads. |
| `FlushStream()` | Ensures values written by prior `Stream` (non-temporal store) calls are visible on the current core. Call after streaming stores before publishing availability to other cores. |
| `FlushCacheline(const void* p)` | Invalidates and flushes the cache line containing `p`. |
| `LoadFence()` | Delays subsequent loads until prior loads are visible. |
| `Pause()` | Hints that you are inside a spin loop; reduces power consumption and coherency traffic. |
| `StreamCacheLine(from, to)` | Copies one aligned cache line from `from` to `to`, bypassing the cache. |

**Example:**

```cpp
#include "hwy/cache_control.h"

void PrefetchRegion(const float* data, size_t count) {
  constexpr size_t kCacheLineBytes = 64;
  for (size_t i = 0; i < count * sizeof(float); i += kCacheLineBytes) {
    hwy::Prefetch(reinterpret_cast<const char*>(data) + i);
  }
}
```

## hwy/nanobenchmark.h

Precisely measures elapsed time for small to medium regions of code. Unlike typical microbenchmark frameworks that repeat the same input, `nanobenchmark` varies the input according to a user-specified distribution for more realistic branch-prediction behavior. Precision is approximately 0.2%.

```cpp
#include "hwy/nanobenchmark.h"
```

Key types and functions:

| Name | Description |
|------|-------------|
| `hwy::FuncInput` | `size_t` — the input argument type passed to the measured function. |
| `hwy::FuncOutput` | `uint64_t` — the "proof of work" returned by the measured function to prevent elision. |
| `hwy::Result` | Holds `input`, `ticks` (robust duration estimate), and `variability` (relative MAD). |
| `hwy::Params` | Configuration: precision, subset ratio, time budget, verbosity. |
| `hwy::Measure(func, arg, inputs, n, results, params)` | Measures `func` over `n` inputs. Returns the number of results written. |
| `hwy::MeasureClosure(closure, inputs, n, results, params)` | Convenience wrapper for lambdas with captures. |

**Example:**

```cpp
#include "hwy/nanobenchmark.h"

namespace {

hwy::FuncOutput BenchSIMD(const void* /*unused*/, hwy::FuncInput size) {
  // ... SIMD kernel operating on `size` elements ...
  return hwy::Unpredictable1();  // prevent elision
}

}  // namespace

void RunBenchmark() {
  const hwy::FuncInput inputs[] = {128, 256, 512, 1024};
  hwy::Result results[4];

  const size_t num_results = hwy::Measure(
      &BenchSIMD, /*arg=*/nullptr, inputs, /*num_inputs=*/4, results);

  for (size_t i = 0; i < num_results; i++) {
    printf("size=%zu: %.1f ticks (variability %.3f)\n",
           results[i].input, results[i].ticks, results[i].variability);
  }
}
```

## hwy/print-inl.h

Defines `Print()` for writing vector lanes to `stderr`. Because this is an `-inl.h` header, it must be included inside `HWY_NAMESPACE` (between `HWY_BEFORE_NAMESPACE()` and `HWY_AFTER_NAMESPACE()`).

```cpp
// Inside HWY_NAMESPACE only:
#include "hwy/print-inl.h"
```

**Signature:**

```cpp
// Prints caption followed by up to max_lanes lanes starting at index lane.
V Print(D d, const char* caption, V v,
        size_t lane = 0, size_t max_lanes = 7);
```

**Example:**

```cpp
#include "hwy/highway.h"
#include "hwy/print-inl.h"   // inside HWY_NAMESPACE

HWY_BEFORE_NAMESPACE();
namespace HWY_NAMESPACE {

namespace hn = hwy::HWY_NAMESPACE;

void DebugKernel() {
  const hn::ScalableTag<float> d;
  auto v = hn::Iota(d, 1.0f);
  hn::Print(d, "v = ", v);     // prints: v = 1, 2, 3, 4, ...
}

}  // namespace HWY_NAMESPACE
HWY_AFTER_NAMESPACE();
```

`Print` is also available when `hwy/tests/test_util-inl.h` has been included.

## hwy/tests/test_util-inl.h

Provides macros for running Highway tests across all available targets, plus per-target helper functions. Include this header in test files only — it is not part of the production API.

```cpp
// In test files only — must be inside HWY_NAMESPACE.
#include "hwy/tests/test_util-inl.h"
```

Key macros:

| Macro | Description |
|-------|-------------|
| `HWY_BEFORE_TEST(suite)` | Declares a test suite that iterates over all enabled targets. |
| `HWY_EXPORT_AND_TEST_P(suite, func)` | Registers `func` as a parameterized test in `suite`. |
| `HWY_AFTER_TEST()` | Closes the test suite declaration. |

**Example:**

```cpp
#undef HWY_TARGET_INCLUDE
#define HWY_TARGET_INCLUDE "my_op_test.cc"
#include "hwy/foreach_target.h"
#include "hwy/highway.h"
#include "hwy/tests/test_util-inl.h"

HWY_BEFORE_NAMESPACE();
namespace HWY_NAMESPACE {

namespace hn = hwy::HWY_NAMESPACE;

void TestAdd(/* HWY test framework passes target info */) {
  const hn::ScalableTag<float> d;
  auto a = hn::Set(d, 1.0f);
  auto b = hn::Set(d, 2.0f);
  auto c = hn::Add(a, b);
  HWY_ASSERT_EQ(3.0f, hn::GetLane(c));
}

HWY_EXPORT_AND_TEST_P(MyTest, TestAdd);

}  // namespace HWY_NAMESPACE
HWY_AFTER_NAMESPACE();
```
